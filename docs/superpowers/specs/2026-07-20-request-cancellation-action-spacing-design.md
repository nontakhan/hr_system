# ปรับระยะห่างปุ่มยกเลิกในหน้าคำขอ

## เป้าหมาย

ปรับการแสดงสถานะและปุ่ม `ยกเลิก` หรือ `ขอยกเลิก` ในตารางประวัติคำขอไม่ให้ชิดติดกัน และให้ทุกหน้าคำขอใช้รูปแบบเดียวกันทั้งหน้าจอเดสก์ท็อปและจอแคบ

## ขอบเขต

- ประวัติการลา
- ประวัติคำขอมาสาย ออกก่อน และทำงานล่วงเวลา
- ประวัติคำขอสลับวันหยุด
- ประวัติคำขออบรมหรือกิจกรรม
- ตรวจค้นจุดแสดงปุ่มยกเลิกอื่นที่ใช้รูปแบบเดียวกันก่อนปิดงาน

ไม่เปลี่ยนสิทธิ์ เงื่อนไขสถานะ ข้อความยืนยัน API หรือกระบวนการอนุมัติการยกเลิก

## การออกแบบ

ใช้ container กลางสำหรับกลุ่มการทำงานภายในเซลล์ตาราง โดยจัดวาง badge สถานะและปุ่มเป็น flex ที่มีช่องว่างคงที่ รองรับการตัดขึ้นบรรทัดใหม่เมื่อพื้นที่ไม่พอ และจัดแนวให้อ่านง่าย ปุ่มยังคงใช้ Bootstrap `btn-sm btn-outline-danger` และรูปแบบ Sarabun เดิม

ข้อความเหตุผลการยกเลิกจะแสดงเป็นบล็อกแยกจากกลุ่มปุ่ม เพื่อไม่เบียด badge หรือปุ่มในแถวเดียวกัน

## การนำไปใช้

- เพิ่มหรือใช้คลาส CSS กลางเพียงชุดเดียว แทนการกำหนด margin เฉพาะรายหน้า
- ปรับ HTML template ใน JavaScript ของทุก workflow ให้วางสถานะและ action ใน container ดังกล่าว
- รักษา escaping ของข้อมูลและ handler ยกเลิกเดิม

## การตรวจสอบ

- เพิ่ม regression test ที่ยืนยันว่าทุก workflow ใช้ layout class กลาง
- รัน test แบบ red-green ก่อนและหลังแก้
- ตรวจ syntax JavaScript ของไฟล์ที่เปลี่ยน
- รัน focused tests ของ workflow การยกเลิก
- ตรวจ `git diff --check` และตรวจว่าไม่ได้แตะไฟล์นอกขอบเขต
- หากสภาพแวดล้อมพร้อม ตรวจหน้าใช้งานจริงที่ความกว้างเดสก์ท็อปและจอแคบ

## เกณฑ์ยอมรับ

- badge สถานะกับปุ่มยกเลิกมีช่องว่างที่มองเห็นชัดเจน
- เมื่อพื้นที่แคบ องค์ประกอบตัดบรรทัดโดยไม่ซ้อนหรือชิดกัน
- ทุกหน้าคำขอในขอบเขตแสดงผลสม่ำเสมอ
- พฤติกรรมยกเลิกและสิทธิ์ผู้ใช้ไม่เปลี่ยน
