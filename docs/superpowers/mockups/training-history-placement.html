ระบบประวัติการฝึกอบรมในหน้า Employee View

เปรียบเทียบตำแหน่งปุ่มและวิธีแสดงประวัติ ก่อนเริ่มทำจริงใน employee_view.php

Option A: ปุ่มข้างโยกย้าย + ตารางประวัติด้านล่าง
ย้อนกลับ อบรม โยกย้าย แก้ไข
ข้อมูลพนักงาน
Card: ประวัติการฝึกอบรม + ปุ่ม refresh / เพิ่ม
แนะนำ

A. ปุ่มบันทึกอบรมข้างโยกย้าย + ตารางประวัติแยก

เข้ากับหน้าปัจจุบันที่สุด เห็นทั้ง action และ history ชัด เหมาะกับ HR ที่เปิดแฟ้มพนักงานแล้วต้องบันทึกหลักสูตรหรือดูย้อนหลังทันที

Option B: เพิ่ม/ดูจาก Card ประวัติเท่านั้น
ย้อนกลับ โยกย้าย แก้ไข
ข้อมูลพนักงาน
Card: ประวัติอบรม มีปุ่ม + เพิ่มอบรม ในหัว card

B. Action อยู่กับ Card ประวัติ

หัวหน้าหน้าไม่แน่น แต่ผู้ใช้ต้องเลื่อนลงถึงจะเพิ่มรายการอบรมได้ เหมาะถ้าอยากให้ปุ่มด้านบนมีเฉพาะ action หลักจริงๆ

Option C: Modal เดียวรวมเพิ่ม/ดูประวัติ
ประวัติอบรม โยกย้าย แก้ไข
เปิด modal แล้วเห็นฟอร์ม + รายการย้อนหลังในหน้าต่างเดียว

C. ทุกอย่างอยู่ใน Modal

เพิ่มไฟล์น้อย แต่ประวัติไม่ visible ในหน้าแฟ้มพนักงาน และถ้ารายการอบรมเยอะจะอ่านยากกว่า

ข้อมูลที่ควรเก็บในรอบแรก

วันที่อบรม, ชื่อหลักสูตร, หน่วยงาน/ผู้จัด, ประเภทอบรม, ผลลัพธ์/สถานะ, วันหมดอายุใบรับรองถ้ามี, หมายเหตุ และผู้บันทึก
